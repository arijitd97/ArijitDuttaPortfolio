Twitter Simulator

A Twitter simulation built using Akka Model in F# where there are many users performing operations simultaneously. A peer to peer system that is capable of storing data in nodes without having to keep a central directory. The system has support for automatic error resolution in case of node failure or if nodes join or leave the system. The system allows efficient searching and storing data reducing complexity to log(n) using the chord protocol.

Gossip and Push Sum Protocol

Implemented Gossip Protocol and the Push Sum protocol using the Actor Model in F# and analyzed the runtimes. Comparison of the runtimes between various topologies were done to check which topology works better. Topologies like Line, Full, 2D and 3D were used. These systems are used for communication between unreliable nodes having very less power. Objective is to check if the systems converge using the given topologies and the protocols.

F#, Communication in large scale Distributed Systems
